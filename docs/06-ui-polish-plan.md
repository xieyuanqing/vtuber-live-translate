# UI 整理方案（v2.0.1+）

## 目标

设计风格保持 Luminous Blue，把当前「功能可用但偏表单」的界面，整理成更接近 `ui.zip` 基线的 iOS 工具感。

原则：

1. **功能不丢**：翻译内核、权限、快照、历史、AI 分析不变。
2. **心智更简单**：用户最终选的是「方案」，不是分别点场景和术语库。
3. **数据据实**：不展示后端没有的假字段（词条数、最后更新时间、多语言 chips）。

## 概念整理：场景 + 术语库 → 方案

| 概念 | 代码 | 用户怎么理解 |
|---|---|---|
| 方案 | `TranslationPlan` + 可选 `glossaryKey` | 一次翻译用什么配置 |
| 场景 | `scenePresetId` | 方案内部的风格预设 |
| 术语库 | `GlossaryProfile` | 方案可绑定的专名资料 |
| 本场背景 | `SessionPromptContext` | 只对这一场生效，不进长期方案 |

UI 呈现：

- 主页只显示「当前方案」摘要：`场景 · 语言 · 术语库（如有）`
- 点方案卡 → 底部弹窗 / 后续方案库列表
- 术语库仍可在设置里维护，但日常路径是「嵌在方案里选」

## 分期

### 第一期：主页视觉（本轮）

同传 / 视频主页：

1. 语言改成居中胶囊区（保留现有下拉绑定）
2. 同传：大圆主按钮 + 状态胶囊 + 音量
3. 视频：全宽圆角主按钮 + 状态
4. 「当前方案」摘要卡（点开原 bottom sheet）
5. 当前字幕预览卡保留

不动：CaptureService、PromptBuilder、权限流、历史结构。

### 第二期：方案库列表

- 新增「自定义方案」列表页（segmented：同传/视频）
- 卡片：名称、场景、语言、是否绑定术语库
- FAB：新建方案
- 点卡片：编辑 / 应用

### 第三期：术语库列表化

- 列表 + 搜索 + 进入详情编辑
- 设置入口改为列表页，不再一页塞完整表单

### 第四期：细节

- 悬浮胶囊底栏更圆
- 场景图标网格（若仍需要快捷切换）
- 涟漪动画

## 本轮不做

- 不改翻译协议 / setup JSON
- 不发明词汇数统计、更新时间
- 不合并删除 Glossary 数据模型（只改呈现与入口）
